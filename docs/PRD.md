## 1. 제품 개요

팀이 협업하여 업무를 시각적으로 관리하는 칸반 보드 웹 애플리케이션.  
v2.0에서는 실제 소셜 로그인(Google · GitHub)과 이메일 인증을 통해 사용자 계정을 관리하며, 공유 DB를 통해 여러 팀원이 동일한 보드를 실시간으로 사용할 수 있다.

---

## 2. 목표 사용자

| 구분 | 설명 |
|------|------|
| 개인 사용자 | 자신의 업무를 칸반 방식으로 관리 |
| 팀 Admin | 팀원 역할을 관리하고 전체 카드를 제어 |
| 팀 Member | 자신에게 배정된 카드를 관리 |

## 3. 기능 요구사항

### AUTH — 인증 (신규 v2.0)

| ID | 요구사항 | 우선순위 |
|----|----------|---------|
| AUTH-01 | Google 소셜 로그인 (OAuth 2.0) | P0 |
| AUTH-02 | GitHub 소셜 로그인 (OAuth 2.0) | P0 |
| AUTH-03 | 이메일 + 비밀번호 로그인 | P0 |
| AUTH-04 | 이메일 + 비밀번호 회원가입 (이름 포함) | P0 |
| AUTH-05 | 로그아웃 | P0 |
| AUTH-06 | 미인증 사용자는 보드 화면에 접근할 수 없어야 함 | P0 |
| AUTH-07 | 최초 가입자는 자동으로 Admin 역할 부여 | P1 |
| AUTH-08 | 이메일 인증 (Supabase 설정에 따라 선택 적용) | P2 |

### BOARD — 칸반 보드

| ID | 요구사항 | 우선순위 |
|----|----------|---------|
| BOARD-01 | 3개 컬럼(To-do, In Progress, Done) 표시 | P0 |
| BOARD-02 | 각 컬럼 카드 수 표시 | P1 |
| BOARD-03 | 반응형 레이아웃 (960px 이하 2열, 640px 이하 1열) | P1 |

### CARD — 카드 관리

| ID | 요구사항 | 우선순위 |
|----|----------|---------|
| CARD-01 | 카드 생성 (제목, 설명, 담당자) | P0 |
| CARD-02 | 카드 수정 | P0 |
| CARD-03 | 카드 삭제 (확인 다이얼로그) | P0 |
| CARD-04 | 컬럼 간 드래그 앤 드롭 이동 | P0 |
| CARD-05 | 컬럼 내 순서 변경 (드래그 앤 드롭) | P0 |
| CARD-06 | 카드에 담당자 아바타 + 이름 표시 | P1 |
| CARD-07 | Admin은 모든 카드 수정/이동 가능 | P0 |
| CARD-08 | Member는 본인 담당 카드만 수정/이동 가능 | P0 |
| CARD-09 | Admin은 담당자를 자유롭게 변경 가능 | P0 |
| CARD-10 | Member는 담당자를 본인으로만 설정 | P0 |

### USR — 사용자 · 팀원

| ID | 요구사항 | 우선순위 |
|----|----------|---------|
| USR-01 | 로그인한 사용자 정보(이름, 아바타, 역할)를 헤더에 표시 | P0 |
| USR-02 | 전체 팀원 목록 조회 (인증된 사용자 전체) | P1 |
| USR-03 | Admin은 팀원의 역할(Admin ↔ Member)을 변경할 수 있다 | P1 |
| USR-04 | 팀원 삭제 시 해당 카드의 담당자는 미배정(null)으로 변경 | P1 |

### DEPLOY — 배포 (신규 v2.0)

| ID | 요구사항 | 우선순위 |
|----|----------|---------|
| DEPLOY-01 | GitHub Pages에 자동 배포 | P0 |
| DEPLOY-02 | main 브랜치 kanban 파일 변경 시 자동 배포 트리거 | P0 |

## 4. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 보안 | Row Level Security(RLS)로 DB 레벨 권한 강제 |
| 보안 | XSS 방지: 사용자 입력은 `textContent`로만 DOM에 삽입 |
| 보안 | HTTPS 전용 배포 (GitHub Pages 기본) |
| 가용성 | 외부 서버 없이 Supabase CDN + GitHub Pages로만 운영 |
| 성능 | 초기 로드 시 카드 전체 1회 fetch, 이후 로컬 상태 관리 |
| UX | 로딩 인디케이터, 에러 toast, 빈 상태 안내 메시지 제공 |
| 접근성 | aria-label, role 속성 주요 인터랙티브 요소에 적용 |
| 호환성 | 최신 Chromium · Firefox · Safari (ES2020+ 지원 브라우저) |
