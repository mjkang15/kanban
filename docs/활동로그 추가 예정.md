# 활동 로그 기능 추가 계획

## 1. DB — `activity_logs` 테이블 추가 (`supabase-setup.sql`)

```sql
CREATE TABLE public.activity_logs (
    id          uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
    actor_id    uuid        REFERENCES public.profiles(id) ON DELETE SET NULL,
    card_id     uuid        REFERENCES public.cards(id) ON DELETE SET NULL,
    action_type text        NOT NULL,
    payload     jsonb       DEFAULT '{}',
    created_at  timestamptz DEFAULT now()
);

-- RLS: 인증 유저 전체 조회, 본인만 삽입
ALTER TABLE public.activity_logs ENABLE ROW LEVEL SECURITY;
CREATE POLICY "logs_select" ON public.activity_logs FOR SELECT TO authenticated USING (true);
CREATE POLICY "logs_insert" ON public.activity_logs FOR INSERT TO authenticated WITH CHECK (actor_id = auth.uid());
```

### 기록할 액션 타입

| action_type | 설명 |
|---|---|
| `card_created` | 카드 생성 |
| `card_updated` | 제목/설명/담당자 수정 |
| `card_moved` | 컬럼 간 이동 (드래그) |
| `card_deleted` | 카드 삭제 |
| `role_changed` | 팀원 역할 변경 |

## 2. JS — `script.js` 변경

### `logActivity()` 함수 추가

```js
async function logActivity(actionType, cardId, payload = {}) {
    await sb.from('activity_logs').insert({
        actor_id:    authUser.id,
        card_id:     cardId || null,
        action_type: actionType,
        payload,
    });
}
```

### 각 함수에 로그 삽입

| 함수 | 변경 위치 | 기록 내용 |
|---|---|---|
| `createCard()` | 성공 후 | `card_created`, 제목·컬럼 |
| `updateCard()` | 성공 후 | `card_updated`, 변경 전/후 diff |
| `deleteCardById()` | 삭제 전 카드 정보 캡처 후 | `card_deleted`, 제목 |
| `handleDrop()` | `syncCardOrders` 호출 후 | `card_moved`, 원본→목적 컬럼 |
| `updateProfileRole()` | 성공 후 | `role_changed`, 대상 이름·역할 |
