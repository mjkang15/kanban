## 1. 앱 초기 로드 흐름

```
앱 로드 (index.html)
    │
    ▼
Supabase onAuthStateChange 등록
    │
    ├─ [세션 있음]
    │       ▼
    │   프로필 fetch → 카드 fetch → 보드 화면 표시
    │
    └─ [세션 없음]
            ▼
        인증 화면 표시
```

## 2. 인증 흐름

### 2.1 Google / GitHub OAuth 로그인

```
인증 화면
    │
    └── [Google로 계속] 또는 [GitHub로 계속] 클릭
            │
            ▼
        Supabase OAuth 리다이렉트
        (Google / GitHub 로그인 페이지)
            │
            ├─ [인증 성공]
            │       ▼
            │   앱으로 리다이렉트 (redirectTo URL)
            │       ▼
            │   onAuthStateChange 'SIGNED_IN' 이벤트
            │       ▼
            │   신규 사용자 → DB 트리거 → profiles 자동 생성
            │   기존 사용자 → profiles 조회
            │       ▼
            │   보드 화면 표시
            │
            └─ [인증 취소/실패]
                    ▼
                인증 화면 유지
                에러 toast 표시
```

### 2.2 이메일 로그인

```
인증 화면 > [로그인] 탭
    │
    └── 이메일 + 비밀번호 입력 → [로그인] 클릭
            │
            ├─ [성공]
            │       ▼
            │   onAuthStateChange 'SIGNED_IN'
            │       ▼
            │   보드 화면 표시
            │
            └─ [실패: 잘못된 자격증명]
                    ▼
                에러 메시지 표시 (인라인)
```

### 2.3 이메일 회원가입

```
인증 화면 > [회원가입] 탭
    │
    └── 이름 + 이메일 + 비밀번호 입력 → [회원가입] 클릭
            │
            ├─ [이메일 확인 OFF] (개발 환경)
            │       ▼
            │   즉시 세션 생성 → onAuthStateChange
            │       ▼
            │   DB 트리거 → profiles 자동 생성
            │       ▼
            │   보드 화면 표시
            │
            ├─ [이메일 확인 ON] (프로덕션)
            │       ▼
            │   "확인 이메일을 발송했습니다" 안내
            │       ▼
            │   이메일 링크 클릭 → 세션 생성 → 보드 표시
            │
            └─ [유효성 오류]
                    ▼
                에러 메시지 표시 (인라인)
```

## 4. 카드 관리 흐름

### 4.1 카드 추가

```
[+ 카드 추가] 클릭 (특정 컬럼)
    │
    ▼
카드 모달 열기
    │
    ├── 제목 입력 (필수)
    ├── 설명 입력 (선택)
    └── 담당자 선택
          ├─ [Admin] 드롭다운: 미배정 + 전체 팀원
          └─ [Member] 고정: 본인 (변경 불가)
    │
    └── [저장] 클릭
              │
              ├─ 제목 없음 → 에러 강조 + shake 애니메이션
              └─ 정상 입력
                      ▼
                  Supabase INSERT
                      ▼
                  카드 목록 갱신 → 보드 재렌더링
```

### 4.2 카드 수정

```
카드의 [✏ 수정] 버튼 클릭 (권한 있는 경우만 표시)
    │
    ▼
카드 모달 열기 (기존 값 채워짐)
    │
    └── 수정 후 [저장]
              ▼
          Supabase UPDATE
              ▼
          보드 재렌더링
```

### 4.3 카드 삭제

```
카드의 [🗑 삭제] 버튼 클릭 (권한 있는 경우만 표시)
    │
    ▼
confirm 다이얼로그
    │
    ├─ [취소] → 아무 변화 없음
    └─ [확인]
              ▼
          Supabase DELETE
              ▼
          보드 재렌더링
```

### 4.4 드래그 앤 드롭

```
카드 드래그 시작 (권한 확인)
    │
    ├─ [권한 없음] → 드래그 취소
    └─ [권한 있음]
              │
              ▼
          다른 컬럼 또는 같은 컬럼에 드롭
              │
              ├─ 드롭 위치 계산 (마우스 Y 기준)
              └─ 보드 DOM 업데이트 (즉시 반영)
                      ▼
                  Supabase UPSERT (column_name, sort_order 일괄 저장)
                      ▼
                  보드 재렌더링 (DB 순서와 동기화)
```

## 5. 팀원 관리 흐름 (Admin 전용)

### 5.1 팀원 패널 열기

```
헤더 [팀 관리] 버튼 클릭 (Admin만 표시)
    │
    ▼
Supabase profiles 전체 조회
    │
    ▼
팀원 패널 슬라이드인 (우측)
각 팀원: 아바타, 이름, 역할 뱃지, [역할 변경] 버튼
```

### 5.2 역할 변경

```
팀원 패널에서 특정 팀원의 [→관리자] 또는 [→팀원] 클릭
    │
    ▼
Supabase UPDATE profiles SET role = ?
    │
    └─ [성공] → 팀원 패널 재렌더링
    └─ [실패] → 에러 toast
```

## 6. 권한 분기 요약

| 상황 | Admin | Member |
|------|-------|--------|
| 카드 드래그 | 모든 카드 | 본인 담당 카드만 |
| 수정/삭제 버튼 표시 | 모든 카드 | 본인 담당 카드만 |
| 담당자 선택 드롭다운 | 전체 팀원 선택 가능 | 비활성화 (본인 고정) |
| 팀 관리 버튼 표시 | ✅ | ❌ (hidden) |
| 역할 변경 버튼 표시 | ✅ (본인 제외) | ❌ |

## 7. 에러 처리 흐름

```
Supabase API 호출
    │
    ├─ [성공] → UI 갱신
    │
    └─ [에러]
          ▼
      showToast(message, 'error')
          ▼
      3초 후 자동 제거
```

인증 에러 코드별 한국어 메시지:
- `invalid_credentials` → "이메일 또는 비밀번호가 올바르지 않습니다."
- `email_not_confirmed` → "이메일 인증이 필요합니다. 메일함을 확인해 주세요."
- `user_already_exists` → "이미 가입된 이메일입니다."
- 그 외 → "오류가 발생했습니다: {message}"
